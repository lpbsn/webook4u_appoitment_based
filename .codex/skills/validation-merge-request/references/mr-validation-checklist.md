# Merge Request Validation Checklist

## Artifact chain
- feature brief present when needed
- functional spec present when needed
- technical spec or technical story present
- delivery-ready ticket present
- hardening review present

## Conformity
- implementation matches the approved objective
- scope matches the ticket
- non-goals remain respected

## Evidence
- required tests and checks are present
- claimed completion matches actual proof

## Final blockers
- contradiction between artifact chain and implementation
- missing required proof
- unresolved critical risk
