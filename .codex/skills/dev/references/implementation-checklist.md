# Implementation Checklist

## Before editing
- Confirm the ticket objective is singular and explicit.
- Identify impacted files or layers.
- Identify required tests.
- Identify explicit non-goals.

## During implementation
- Keep the diff aligned with the ticket.
- Surface contradictions immediately.
- Avoid silent refactors outside scope.

## Before closing the work
- Verify behavior against the ticket.
- Verify expected tests or checks.
- Note any residual risk or limitation.
